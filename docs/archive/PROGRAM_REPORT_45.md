## IF/THEN Branches Taken (K2)

1. **Server identity collision on port 8000.** An early `curl localhost:8000/openapi.json` (before I'd chosen a dedicated port) returned `/comms/ingest`, `/comms/mappings/confirm`, `/comms/pending-mappings/{notion,slack}` routes that should not exist on fresh `origin/main` — grep of `api.py`'s actual `@app.get`/`@app.post` decorators confirmed only 12 real routes, none of them `/comms/*`. Root cause: a pre-existing leftover process (PID 68880, started before this dispatch began, bound `127.0.0.1:8000`) intercepted the request. Chose: kill only my own conflicting instance, never touch the other PID, move to a dedicated port (8010, later 8012 after the port-mapping cleanup below). Lesson recorded in PROGRESS.md: verify server identity before trusting a curl response on a well-known port, on a shared dev box with many concurrent sessions.
2. **`commsRowText`/`CommsDrawer` don't exist.** The dispatch's Mission, Reading list, Decisions, and Routing table all treat `commsRowText` as an existing pattern to mirror. Fresh full-repo grep: zero matches. Comms is dormant (`briefField: null`), same category as Calendar/Agent-history — not a working humanized precedent. Per K1 (disk > dispatch prose), mirrored `StakeholderPersonRow`'s real, working two-register pattern instead (the exact same file, same commit, functionally equivalent target). Not a STOP: the humanization plan itself remained fully applicable, just retargeted to a different (better, because real) existing example.
3. **Only 4 generic-Drawer targets, not 5.** `DRAWERS` currently has Stakeholders already humanized via its own `StakeholderDrawer` (landed the same Harvest-17 commit this worktree branches from). The actual remaining `JSON.stringify(row)`-rendering targets are Onboarding/Telemetry/Success plan/Cases — 4, not 5. Commit message and this report state the corrected scope explicitly rather than silently claim "5 drawers."
4. **Formatter prop typed `unknown` + explicit casts, not `any`/`never`.** A naive `(row: never)` signature failed typecheck (`unknown` not assignable to `never` at the `.map()` call site). Tried `any` next; reverted because this repo's `.eslintrc.json` only extends `next/core-web-vitals` (no `@typescript-eslint` plugin registered), so an `eslint-disable` comment for a nonexistent rule itself errors. Settled on `(row: unknown) => {...}` (matching this file's existing `Brief = Record<string, unknown>` idiom) with an explicit cast at each of the 4 `DRAWERS` assignment sites — the standard TypeScript pattern for a heterogeneous callback table.
5. **CORS/port/auth environment setup for live verification.** Three env issues discovered and fixed, none requiring an edit to `api.py`/`next.config.mjs` (outside ownership): (a) `package.json`'s `dev` script hardcodes `NEXT_PUBLIC_API_BASE=http://localhost:8000` ahead of `next dev`, silently overriding any externally-set value of the same name — invoked `next dev` directly via `node_modules/.bin/next` instead of through `npm run dev`; (b) `api.py`'s CORS `allow_origins` is hardcoded to `http://localhost:3000` only — ran the UI dev server on port 3000 specifically rather than editing CORS; (c) `/sweep` calls `_require_write_auth`, returning 401 without a token — enabled `ULTRA_CSM_DEMO_NOAUTH=1`, the pre-existing, documented local-demo convention (matching the already-existing `ops-surface-api` entry in the shared `~/.claude/launch.json`, not invented). Appended 2 new named launch.json entries (`ui-data-depth-api`, `ui-data-depth-ui`) rather than overwriting the pre-existing entries other concurrent sessions may depend on.
6. **Candidate-selection re-scoping (the big one — see Owner Asks).** Reading `docs/SYNTHETIC_UNIVERSE_BIBLE.md` in full before seeding (per the dispatch's own reading-list instruction) surfaced that the dispatch's headline account (Ironhorse Freight Co) and every other queue-visible account at day 140 is one of the bible's calibrated sets (6 arcs, red herrings, 27 boring controls — Ironhorse is explicitly one of these 27 — tier-mirror/proof accounts, or the 25-account cohort-truth set). Seeding any of them would risk corrupting the exact eval-battery assertion each is calibrated for. Chose 4 accounts (Cobalt Fleet Ops, Northbend Haulage, Cedarfield Industrial Supply, Elmwood Trucking) that clear every named list and aren't tech-touch (the bible's own "thinness is correct at tech tier" design law). This means the seeded accounts are NOT the ones a demo queue-walk actually visits at day 140 — disclosed explicitly in Owner Asks rather than silently resolved either direction.
